> 김영한님의 JPA 로드맵을 따라 학습하면서 정리한 노트입니다.

## 영속성 컨텍스트

JPA를 이해하기 위한 가장 핵심적인 개념으로, "엔티티를 영구 저장하는 환경"을 의미합니다. `EntityManager` 내부에 존재하며 `EntityManager.persist(entity)` 메서드를 통해 접근합니다.

## 엔티티의 생명주기

- **비영속 (new/transient)**: 영속성 컨텍스트와 무관한, 새로 생성된 객체 상태입니다.
- **영속 (managed)**: 영속성 컨텍스트에서 관리되는 상태입니다. `persist()` 호출 시 진입하지만, 실제 DB 쿼리는 트랜잭션 커밋 시점에 발생합니다.
- **준영속 (detached)**: 영속성 컨텍스트에서 분리된 상태로, `detach()`, `clear()`, `close()` 메서드로 전환됩니다.
- **삭제 (removed)**: `remove()` 메서드로 DB 삭제를 요청한 상태입니다.

## 영속성 컨텍스트의 주요 이점

- **1차 캐시**: `Map` 구조로 DB PK를 키, 객체를 값으로 저장하여 조회 성능을 향상시킵니다.
- **동일성 보장**: 같은 트랜잭션 내에서 동일한 엔티티 조회 시 동일성(`==`)을 보장합니다.
- **쓰기 지연**: "여러 엔티티 변경사항이 메모리에 축적되었다가 커밋 시점에 일괄 반영됩니다."
- **변경 감지**: 엔티티 수정 시 자동으로 SQL을 생성하여 명시적 업데이트 호출이 불필요합니다.

## Flush (플러시)

영속성 컨텍스트의 변경사항을 데이터베이스에 동기화하는 작업입니다. 1차 캐시는 유지되며, 쓰기 지연 SQL 저장소의 쿼리들이 DB로 전송됩니다.

**발생 시점**

- `em.flush()` 직접 호출
- 트랜잭션 커밋
- JPQL 쿼리 실행

**플러시 모드**

- `FlushModeType.AUTO` (기본값, 커밋·쿼리 실행 시)
- `FlushModeType.COMMIT` (커밋만)

중요한 점은 "플러시는 영속성 컨텍스트를 비우는 게 아니라 변경사항을 DB와 동기화하는 작업"이라는 것입니다.
